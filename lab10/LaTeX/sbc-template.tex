\section{Parte 2 - Perguntas}

Nessa questão eu testei alguns vetores ordenados inversamentes e crescente e aleatórios.
Pois se eu fosse me basear diretaemente somente por tamanho do vetor sempre teríamos que com 1k é melhor e 50k seria pior.\\

 Para cada método de ordenação. Qual foi seu melhor e pior caso observado?\\
\textcolor{blue}{BubleSort}\\
\textcolor{blue}{Melhor caso é quando o vetor ja estava ordenado.}\
\textcolor{blue}{E pior caso é quando o vetor estava ordenado na ordem inversa.}\\

\textcolor{blue}{SelectionSort}\\
\textcolor{blue}{Melhor caso é quando o vetor ja estava ordenado.}\
\textcolor{blue}{E pior caso é quando o vetor encontra-se numa aleatoriedade muito grande, números a serem comparados muitos distantes dos valores da proxima posição}\\

\textcolor{blue}{InsertionSort}\\
\textcolor{blue}{Melhor caso é quando o vetor ja estava ordenado.}\
\textcolor{blue}{E pior caso é quando o vetor encontra-se ordenado inversamente}\\

\textcolor{blue}{HeapSort}\\
\textcolor{blue}{Melhor caso é quando o vetor encontra-se ordenaddo inversamente, ao contrário do insertion que seu pior caso é esse de está ordenado inversamente}\\
\textcolor{blue}{Mais uma vez sendo o inverso do insertion, seu pior caso é quando o vetor está ordenado de forma crescente}\\

\textcolor{blue}{MergeSort}\\
\textcolor{blue}{Melhor caso foi quando o vetor já se encontrou parcialmente ordenado e não precisou fazer muitas trocas}\\
\textcolor{blue}{Pior caso foi quando vetor encontrou muita aleatoriedade. Onde há muitas trocas}\\

\textcolor{blue}{QuickSort}\\
\textcolor{blue}{Melehor caso foi quando ordenou vetores grandes}\\
\textcolor{blue}{Pior caso foi quando os vetores eram pequenos, o insertion estava melhor que o quick para vetores pequenos.}\\


Qual função melhor descreve o desempenho de cada método? 

Abaixo a complexidade assintótica de cada método.\\
\textcolor{blue}{BubleSorte $0(N^2)$}\\
\textcolor{blue}{Selection $0(N^2)$}\\
\textcolor{blue}{Insertion $0(N^2)$}\\
\textcolor{blue}{MergeSort $0(nlogn)$}\\
\textcolor{blue}{HeapSort $0(nlogn)$}\\
\textcolor{blue}{QuickSort $0(nlogn)$}\\

Em quais métodos(e quantidade de dados nos vetores), o número de comprações é um bom substituto para o tempo de execução (ou seja, as duas métricas dão resultados relativos parecidos)?\\
\textcolor{blue}{Como podemos observar na tabela, é evidente que seja o QuickSort.}\\
De forma geral, quais os melhores algoritmos dentre os testados?\\
\textcolor{blue}{QuickSort possui o melhor desempenho, HeapSort segundo, MergeSort terceiro e BubleSort como o pior entre todos.}\\
